Sealion docs

React frontend, C API backend, Postgres database.

The default Sealion UI uses React in a container while the framework keeps auth, sessions, and data access in C.

Runtime Model

browser -> frontend container -> /api proxy -> backend C container -> Postgres

The frontend is the public entrypoint. It proxies /api and /health to the backend so browser requests stay same-origin.

Generated Frontend

React owns page flow, forms, and dashboard rendering. The C backend owns JSON responses, HttpOnly cookies, sessions, validation, and Postgres access.

Styling

The starter uses plain checked-in CSS. Tailwind remains optional for apps that choose it, but it is not required by the framework starter.

Regression Requirements

  • Generated apps include frontend, backend, and database containers.
  • The frontend proxies /api and /health to the backend.
  • Auth uses same-origin cookies without CORS setup.
  • Frontend builds from npm ci using a lockfile.
  • .skin and .scale files are optional, not required by the default starter.
